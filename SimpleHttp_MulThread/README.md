# C语言 epoll+多线程，实现一个单反应堆(单reactor的变体)服务器
## 你现在的架构叫做 “单 Reactor + Thread-Per-Event（一事件一线程）” 模型。
### 1个 epoll（Reactor）： 运行在主线程里，专门负责监听网络事件（收发快递）。
### 多个线程： 每当有客户发来数据，主线程就立刻 pthread_create 临时雇佣一个工人去处理。
## 这个模型的致命弱点：假设瞬间有 10000 个人并发访问你的网页，你的主线程会瞬间调用 10000 次 pthread_create 创建 10000 个线程。服务器的 CPU 会被庞大的“线程上下文切换”瞬间压垮，内存也会爆满。
